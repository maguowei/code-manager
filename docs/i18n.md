# 国际化开发规范

Code Manager 当前随桌面包同步提供简体中文和英文。前端使用 `i18next + react-i18next`，Rust 配置中的 `UiLanguage` 是持久化权威，localStorage 仅用于同步启动缓存。

## 前端文案

- 组件继续通过 `useI18n()` 使用 `language`、`t()` 和异步 `setLanguage()`。
- 词条按 `common/settings/profiles/memory/skills/projects/history/stats/usage/cheatsheet` namespace 维护在 `src/i18n/catalogs/{zh,en}/`。
- key 保持 dotted 形式；带参数消息用 `t(key, { name })`，数量消息用 `_one` / `_other`。
- 不要对翻译结果调用 `.replace()`，不要拼接单位词或用 `profile(s)` 规避复数。

## locale 格式

用户显示的数字、百分比、USD、日期时间和名称排序复用 `src/i18n/format.ts`。`zh` 映射 `zh-CN`，`en` 映射 `en-US`。HTML 日期输入、JSON、日志和持久化值继续使用稳定 ISO 格式。

## 原生文案与 IPC 错误

- 托盘、系统通知和终端聚焦提示集中在 `src-tauri/src/native_i18n.rs`，按 `UiLanguage` 渲染。
- fallible Tauri command 返回 `CommandError { code, args }`。底层错误只进入脱敏日志，不能经 IPC 发送。
- 新错误优先复用现有 `CommandErrorCode`；确需新增时同步 Rust 枚举、`errors.<code>` 中英文词条、bindings 和测试。
- `args` 只允许名称、数量等安全插值，禁止路径、密钥、命令输出和原始 Provider 响应。

## 验证

- 开发中查看报告：`make i18n-audit`
- 提交前严格检查：`make i18n-check`
- 修改 IPC 后：`make bindings && make bindings-check`
- 全量门禁：`make verify`

`i18n-check` 检查中英文 key/占位符、未定义 key、翻译手工替换、固定 locale、用户可见硬编码、原生文案散落和字符串 IPC 错误。代码标识白名单集中在 `scripts/check-i18n.mjs`，每项必须附具体理由。
